关于数据库写入Emoji表情的报错的解决方法

发表于 2018-12-12 阅读次数：

这两天写GooglePlay爬虫的时候发现入库数量与redis中的数量不一致, 翻看lockback日志发现部分插入语句报错，特此记录。

问题描述

在执行sql时， 控制台抛出异常

原因分析

Emoji表情符号为4个字节的字符，而 utf8 字符集只支持1-3个字节的字符，导致无法写入数据库。

解决方案

  • 修改MySQL数据库字符集， 把数据库字符集从utf8 修改为支持1-4 个字节字符的utf8mb4.

若采用java的jdbc来进行入库操作, 则需要额外地升级驱动的版本到5.1.13版本或更高.

以gradle为例(maven类似):

# java # jdbc # mysql
在后台运行 docker 容器会自动退出解决方案
